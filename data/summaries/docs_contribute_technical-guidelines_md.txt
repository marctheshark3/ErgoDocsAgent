This document outlines the technical contribution guidelines for contributing code to Ergo's core repositories. It covers the general workflow for contributing, including finding issues, forking and branching, developing and testing, submitting pull requests, and code review. It also provides troubleshooting guidance for common build and CI issues, as well as dependency management guidelines, particularly around the use of BouncyCastle variants. The document emphasizes the importance of following project-specific practices and collaborating with the community to contribute effectively to the Ergo ecosystem.
