Ergo allows the issuance of one token per transaction, where the token id must match the id of the box of the first input. By iterating over a register used as a nonce, a token with specific id properties can be created, requiring computational work similar to VanityGen-address in Bitcoin. Certain contracts may only accept these Proof-of-Work backed Non-Fungible Tokens (NFTs), and potential applications could include spam filtering.
